What is that?

HotSpot icon menus is a quality plugin for quick popup navigation that is built with love. With this plugin, you can create, dynamically update in seconds cool icons menu that can be attached to any container and work on any platform. Easy to customize, easy to use and very reliable. The plugin includes 400 svg demo icons that are free to use (iconmonstr.com) and are already attached to the plugin and ready to use. We design it to be flexible and smart. Take a look at the Cool demos we created for your admiration.

 

Features:

  1. Quick and smart.
  2. Support different placement and directions.
  3. Animated.
  4. Easy to customize ( no CSS required ~ optional )
  5. Create as many as you want.
  6. 400+ already included icons.
  7. Very flexible - Shapes,Colors, position,shadows.
  8. Easy to attach callbacks functions.
  9. Can be dynamically rebuild and redesign.

Insatallation

To install this plugin just follow these easy three steps. Once finished, it is all about setting a few options when creating a HotSpot icon menu in your page.

 

Step 1:

Extract the purchased hotspot.v.1.0.zip to your main web-site folder. This archive contain 4 folders (doc,js,css,icons). If you already have these folders merge their content.

 

Step 2:

Include the latest jQuery lib, easing.1.3.js, hotspot.v.1.0.min.js and hotspot.v.1.0.css in the HEAD section of your web-page. Just copy paste next lines: 
<head>
<script type="text/javascript" src="hotspot/js/jquery-1.11.0.min.js"></script>
<script type="text/javascript" src="hotspot/js/easing.1.3.js"></script>
<script type="text/javascript" src="hotspot/js/hotspot.1.0.js"></script>
<link href="hotspot/css/hotspot.1.0.css" rel="stylesheet" type="text/css" />
</head>

Creating A HotSpot

All HotSpot icon menu can be created in two ways: By adding an object of options or by creating an HTML list with some specific attributes. Its easy and quick and we recommend while experimenting to enable debug mode to make sure you are setting the menu with the correct options.

 

Basic Facts About HotSpot

  • You can create as many menus as you need.

  • HotSpot requires 4 files: jQuery lib (1.9+), easing.js, hotspot.js, hotspot.css.

  • Menus can be attached to an HTML list or a DIV container.

  • When HotSpot intialize the menu it looks at his parent - If its the BODY the menu will be fixed and if its inside an other DOM element it will be posed insided this container.

Icons

HotSpot menus are based on Icons - Those icons are bound to the plugin using a specific CSS class that allows you to declare the icons to be used easily and make sure that they will be properly displayed on any platform / web-browser.
All icons that are supplied with this plugin are "free to use" and are SVG formatted to make sure they are displayed at the best resolution there is. You can create your own icons or select whatever you need from the already declared icons.

 

Icons classes

Each Icon to be used with this plugin should be declared with CSS - Basically we declare a class name that will be bound to the requested icon. All icons class name when declared need to have those prefix and suffix:
  • .hsi_ - The plugin prefix
  • icon unique name - Your icon unique name
  • _d or _w - Suffix that indicates lightness of the icon (d for dark, w for white)
When setting the icon to be used you don't have to use the hsi_ prefix, just the icons name and the lightness suffix. If you don't add the suffix a dark suffix will be assumed by the plugin, example:
icon:'addfile'  equal to  icon:'hsi_addfile'  equal to  icon:'addfile_d'
"icon" setting is required when declaring the main element ( the first object in the elements array). In the child elements its optional.

Custom Icons

If you want to add your own icons to be used with the plugin you should add a CSS rule as followed: 
/*Dark icon using the _d suffix */
.hsi_yourIconName_d {
    background-image: url( /* path to your icon */ );
    background-position: center center;
    background-repeat: no-repeat;

    /* whatever you want to happen when mouse enter */
}
We recommend using SVG images for your icons just for the sake of resolution.

Events & Methods

HotSpot jQuery plugin has Some basic events and methods that allow you to easily integrate and set its behavior, interact with your page and programmatically interact with each HotSpot instance you created. those methods and events makes HotsSpot a great flexible tool that can be any menu you need: Image tool box, Gallery controls, Media player controls and many more.

 

Events

Each instance has 5 optional Events: created, openCallback, closeCallback, beforeOpen, beforeClose
Those events are set by the option object when creating the menu, they are expecting a function that will be fired by the plugin any time they occurred
Here are some basic examples:
EventBehaviorExample SyntaxExample
created Will execute a function after the plugin finishes the menu creation 
created:function(){ alert("created a hotspot"); }
Will pop-up a message after hotspot is created.
openCallback will trigger a callback function when the menu was opened 
openCallback:function(){ alert("you opened me"); }
openCallback will trigger a callback function when the menu was closed 
closeCallback:function(){ alert("you closed me"); }
beforeOpen will trigger a custom function before menu opens 
beforeOpen:function(){ alert("you are about to open me"); }
beforeClose will trigger a custom function before menu closes 
beforeClose:function(){ alert("you are about to close me"); }

 

To target the current hotspot menu inside the callback function you should use   $(this)  .
Those events can't be defined directly when using a "inline" declaration of the menu (using a List) in this case you still need to declare those events through an object.

 

 

Methods

Each instance created has 6 methods: .destroy_hotspot(), .rebuild_hotspot(), .enable_hotspot(), .disable_hotspot(), .open_hotspot(), .close_hotspot()
Those methods can be triggered from any HotSpot element that was created, they enable you to control the menu behavior and rebuild it dynamically.

 

.destroy_hotspot() :
SyntaxArgsWhat it does
   var hotspot = $('#hotspot').hotspot({ /* your settings */ });
   $(hotspot).destroy_hotspot();
No Args Destroys the Menu and detach all events.

 

.rebuild_hotspot() :
SyntaxArgsWhat it does
   var hotspot = $('#hotspot').hotspot({ /* your settings */ });
   $(hotspot).rebuild_hotspot({ mainSize:40 } /* Optional Settings*/ );
Optional settings object - settings that will be passed will overwrite previous ones. Dynamically Rebuild the menu by passing an object of settings to overwrite the previous ones. If you don't pass any settings the rebuild function will use your previous ones.
If you rebuild the elements option you will need to redefine all the elements - Changing just one element is not supported. A good example of use is the generator we created in this page.

 

.disable_hotspot() :
SyntaxArgsWhat it does
   var hotspot = $('#hotspot').hotspot({ /* your settings */ });
   $(hotspot).disable_hotspot('no-drop',true);
Arg1 {String} - is the cursor (css cursor) to use.
Arg2 {Boolean} - Should the menu close or not. 		Dynamically Disables the menu instance - you can assign a cursor to use when disabled and should the menu close before disabled.

 

.enable_hotspot() :
SyntaxArgsWhat it does
   var hotspot = $('#hotspot').hotspot({ /* your settings */ });
   $(hotspot).enable_hotspot(true);
Arg1 {Boolean /Optional} - Should the menu open or not. By default `False`.
 Dynamically Enable the menu instance - you can set the menu to open after enabled, cursor will become 'pointer' again.

 

.open_hotspot() :
SyntaxArgsWhat it does
   var hotspot = $('#hotspot').hotspot({ /* your settings */ });
   $(hotspot).open_hotspot(true,true);
Arg1 {Boolean /Optional} - Trigger beforeOpen func. Arg2 {Boolean /Optional} - Trigger openCallback func. By default `False`
 Dynamically Open the menu instance.

 

.close_hotspot() :
SyntaxArgsWhat it does
   var hotspot = $('#hotspot').hotspot({ /* your settings */ });
   $(hotspot).close_hotspot(true,true);
Arg1 {Boolean /Optional} - Trigger beforeClose func.
Arg2 {Boolean /Optional} - Trigger closeCallback func.
By default `False` 		Dynamically Close the menu instance.

 

 

Responsive approach

The HotSpot menu can be easily responsive by rebuilding it any time the window size changed. Basically we change the mainSize option when the page is resized - Its good practice to fire a timer That will be reset while the user keeps resizing until he is done, consider this: 
<body>
<div id="hotspot"></div>
<script>
    var myHotspot = $("#hotspot").hotspot({/* options with default main size */});
    var timerResizeHotSpot; /* using a timer to prevent tons of call */
    $(window).resize(function(){
        window.clearTimeout(timerResizeHotSpot);
        timerResizeHotSpot = window.setTimeout(function(){
            var current_width = $(window).width();
            var hotSpotWidth = 80; /* Max Width */
            if (current_width <= 480) hotSpotWidth = 35;
            if (current_width <= 768) hotSpotWidth = 55;
            if (current_width <= 960) hotSpotWidth = 65;
            $(myHotspot).rebuild_hotspot({ mainSize:hotSpotWidth });
        },250);
    });
</script>
</body>
Another thing that can help you positioning your menu relatively to the its parent is using CSS margin with percentage margins margin: { t:"20px" l:"0" r:"40%" b:"0" }  as you can see the margin-right is set to 40% If the hotspot position is set to pagePos:"nw" it will be placed at 40% from the right of the parent width.
